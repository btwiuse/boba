# Boba v0.4: Built-in Middleware

## Goal

Ship the middleware that v0.3's architecture was built for: five concrete middleware implementations that exercise ConnectMiddleware, SessionMiddleware, and layer-3 Middleware end-to-end, plus one new subpackage for Prometheus session metrics and one config knob for initial-resize-timeout protection.

With these in place, boba earns the "no other Go library has layer 2" claim from the v0.3 design note: `osc52gate` is the distinctive middleware that only works because SessionMiddleware wraps the transport-layer OutputReader (a fix that landed in the v0.3 follow-up, `cc10cbda`).

## Design Principles

- **Subpackages per middleware**, under `serve/middleware/<name>/`. Each has its own doc surface, its own options, and its own tests. The `sipmetrics` subpackage lives at `serve/sipmetrics/` to signal its separate dep footprint (Prometheus).
- **Options pattern** for each middleware's public constructor (`New(opts ...Option)`), matching the `serve.With*` options on Server.
- **slog.Logger** as the default logger interface — a modern choice consistent with Go 1.21+ conventions. Logger is always optional; nil or absent falls back to `slog.Default()`.
- **Auto-install follows v0.3 precedent**: `cfg.IdleTimeout > 0` auto-installs the idletimeout SessionMiddleware, just as `cfg.BasicUsername != ""` auto-installs basicAuth in v0.3.
- **YAGNI over completeness.** `osc52gate` prompt mode deferred; per-Sip-message-type metrics deferred; Wish-shaped activeterm replaced with a targeted config knob because boba already enforces the invariant Wish's activeterm guards.

## Scope

Five middlewares + one config knob:

| Component | Layer | Package | Auto-installed? | Purpose |
|---|---|---|---|---|
| `osc52gate` | 2 | `serve/middleware/osc52gate` | No | Clipboard-write escape filter (allow/deny/audit) |
| `idletimeout` | 2 | `serve/builtins.go` (internal) | Yes when `cfg.IdleTimeout > 0` | Close idle sessions |
| `recover` | 3 | `serve/middleware/recover` | No | Catch handler construction panics |
| `logging` | 3 | `serve/middleware/logging` | No | Session start/end structured log |
| `sipmetrics` | 2 | `serve/sipmetrics` | No | Prometheus session metrics |
| `InitialResizeTimeout` | (config) | `serve/config.go` | — | Deadline on initial Resize read |

`idletimeout` is internal-only (not a subpackage) because auto-installation from `NewServer` would create an import cycle with a subpackage that itself imports `serve` for `SessionMiddleware`. This matches the v0.3 precedent for `basicAuth` and `connLimit`. Users who want a custom idle duration set `cfg.IdleTimeout`; users who want different activity semantics (e.g., count output, per-event filters) write their own `SessionMiddleware`.

Replaces `Server.attachIdleTimeout` and the `activity func()` threading that ran through `handleWebSocket` / `handleWebTransport` / `handleInputWS` / `handleInputWT` / `processMessage` / `processWTMessage`. All of that goes away in favor of the `idletimeout` middleware observing inbound activity via the wrapped Session's `InputWriter`.

## Public API Surface

### `serve/middleware/osc52gate`

```go
type Mode int

const (
    ModeAllow Mode = iota
    ModeDeny
    ModeAudit
)

type Option func(*config)

func WithLogger(logger *slog.Logger) Option

// New returns a SessionMiddleware that filters the session's outbound
// byte stream for OSC 52 clipboard-write escapes:
//   ESC ] 52 ; <selection> ; <data> (BEL | ESC \)
//
// Modes:
//   - ModeAllow: explicit no-op, documents the policy decision.
//   - ModeDeny:  strip the escape sequence from the byte stream before
//     it reaches the client.
//   - ModeAudit: pass through unchanged; log the attempt via slog at
//     Info level. The data payload is NOT logged (may contain PII);
//     only selection identifier and payload length.
//
// The scanner is stateful and correctly handles escapes split across
// Read() boundaries.
func New(mode Mode, opts ...Option) serve.SessionMiddleware
```

### `idleTimeoutMiddleware` (internal to serve)

Lives alongside `basicAuthMiddleware` and `connLimitMiddleware` in `serve/builtins.go`:

```go
// idleTimeoutMiddleware returns a SessionMiddleware that closes the
// wrapped Session if no inbound bytes are received for the given
// duration. Inbound means client-to-session writes on InputWriter;
// outbound activity does NOT reset the timer.
//
// The middleware spawns a watchdog goroutine in its constructor; the
// goroutine exits on sess.Done() or when the timer fires and
// sess.Close() has been called.
func idleTimeoutMiddleware(d time.Duration) SessionMiddleware
```

Auto-installed from `NewServer` when `cfg.IdleTimeout > 0`. Unexported because the public configuration surface is `Config.IdleTimeout`; users with different idle semantics write their own `SessionMiddleware`.

### `serve/middleware/recover`

```go
type Option func(*config)

func WithLogger(logger *slog.Logger) Option

// New returns a Middleware that catches panics raised during
// handler(sess) construction. The panic is logged at Error level
// (with stack) via slog, and the wrapped handler returns a tiny
// tea.Model that displays the error and immediately quits.
//
// Runtime panics inside tea.Program.Run are BubbleTea's own concern
// and are not caught by this middleware.
func New(opts ...Option) serve.Middleware
```

### `serve/middleware/logging`

```go
type Option func(*config)

func WithLogger(logger *slog.Logger) Option

// New returns a Middleware that logs session start and end via slog.
// Fields:
//   - "remote_addr"   — the HTTP request remote address (from Session.Context)
//   - "duration_ms"   — wall-clock duration of the handler call, logged at end
//
// The start/end pair brackets the handler invocation, not the entire
// session lifetime — panics or tea.Program errors are reflected in
// the "end" event's fields.
func New(opts ...Option) serve.Middleware
```

### `serve/sipmetrics`

```go
package sipmetrics

type Option func(*config)

func WithRegistry(r prometheus.Registerer) Option  // default: prometheus.DefaultRegisterer
func WithNamespace(ns string) Option               // default: "boba"

// New returns a SessionMiddleware that records Prometheus metrics for
// session lifecycle and byte throughput.
func New(opts ...Option) serve.SessionMiddleware
```

Metrics registered (prefix = namespace, default "boba"):

- `<ns>_sessions_active` — gauge. `+1` when middleware wraps a Session, `-1` when wrapped Close returns.
- `<ns>_session_duration_seconds` — histogram with Prometheus default buckets. Observed at Close.
- `<ns>_session_bytes_received_total` — counter. Incremented by `len(p)` on each InputWriter.Write.
- `<ns>_session_bytes_sent_total` — counter. Incremented by `n` on each OutputReader.Read(p) where `n > 0`.
- `<ns>_session_errors_total` — counter. Incremented when wrapped Close returns a non-nil error.

No labels on the byte counters (per-session cardinality is a Prometheus anti-pattern). Per-Sip-message-type counters are deferred (require a Sip-event hook in core handlers).

### Config knob: `InitialResizeTimeout`

```go
type Config struct {
    // ... existing fields ...

    // InitialResizeTimeout is the maximum time the server will wait
    // for the client's initial Resize message after a WS upgrade or
    // WT CONNECT. Zero or negative means default (10 seconds).
    InitialResizeTimeout time.Duration
}
```

Enforced in `handleWS` (around `conn.Read(ctx)` for the initial Resize) and `handleWT` (around the length-prefixed read sequence) by deriving a `context.WithTimeout` and using it for the pre-session reads. On timeout, close the connection/stream and return before `createSession` is called.

## Architecture

### Directory layout

```
serve/
  middleware/
    osc52gate/
      osc52gate.go
      osc52gate_test.go
    recover/
      recover.go
      recover_test.go
    logging/
      logging.go
      logging_test.go
  sipmetrics/
    sipmetrics.go
    sipmetrics_test.go
  builtins.go              # existing, appends idleTimeoutMiddleware
  builtins_test.go         # existing, appends idletimeout tests
  remoteaddr.go            # new, small, RemoteAddr context helpers
  # other existing files modified: server.go, handlers.go, config.go
```

Each user-installable middleware is a single-file package plus tests. `idletimeout` is internal to the main serve package; its auto-install behavior makes the subpackage layout infeasible (import cycle).

### Consolidation: removing `Server.attachIdleTimeout`

The idletimeout middleware supersedes the existing idle-timeout mechanism. Changes:

1. `Server.attachIdleTimeout` method is deleted.
2. The `activity func()` parameter is removed from these signatures:
   - `handleWebSocket(ctx, conn, sess, opts, debug, activity, cfg)` → drop `activity`.
   - `handleWebTransport(ctx, sess, stream, opts, debug, activity, cfg)` → drop `activity`.
   - `handleInputWS(ctx, conn, sess, opts, debug, activity, cfg, apply)` → drop `activity`.
   - `handleInputWT(ctx, sess, stream, opts, debug, activity, cfg, apply)` → drop `activity`.
3. All `activity()` call sites inside `streamOutputWS`, `streamOutputWT`, `handleInputWS`, `handleInputWT`, `processMessage`, `processWTMessage` are removed.
4. The `attachIdleTimeout` call site in `handleWS`/`handleWT` is removed; the ctx returned by `runLiftedChain` is used directly.
5. `NewServer` auto-appends `idletimeout.New(cfg.IdleTimeout)` to `s.sessionMW` when `cfg.IdleTimeout > 0`, immediately before the basic-auth append in the NewServer body — order among built-ins matters for innermost-first conventions but idletimeout can be installed at any built-in position because it composes cleanly with others.

The net effect: identical user-visible behavior for `cfg.IdleTimeout`, significantly simpler signature graph in `handlers.go`.

### `recover` panic-to-model translation

The wrapped Handler uses named returns and a deferred recover:

```go
func wrapped(sess serve.Session) (m tea.Model, opts []tea.ProgramOption) {
    defer func() {
        if r := recover(); r != nil {
            logger.Error("handler panicked",
                slog.Any("panic", r),
                slog.String("stack", string(debug.Stack())))
            m = panicModel{msg: fmt.Sprintf("%v", r)}
            opts = nil
        }
    }()
    return next(sess)
}
```

`panicModel` is an unexported type in the recover package: `Init() tea.Cmd { return tea.Quit }`, `Update` returns `(self, tea.Quit)`, `View` returns a one-line string containing the panic message. The session ends cleanly within a single tea frame.

### logging middleware and `remote_addr`

`remote_addr` needs to be observable from inside a layer-3 handler. The Session interface does not currently expose it. Two options were considered:

1. **Add `RemoteAddr() string` to the Session interface.** Breaking change to any custom Session implementations.
2. **Propagate via context.** `handleWS`/`handleWT` inject `r.RemoteAddr` into `sess.Context()` via an unexported key; logging middleware reads via a helper.

Option 2 is chosen. The helper `RemoteAddrFromContext(ctx context.Context) string` is exported from the `serve` package so other middleware can use it. If the key is absent (e.g., in unit tests), the helper returns an empty string.

Small supporting change in `serve/server.go`: `handleWS`/`handleWT` add the RemoteAddr to the request context alongside the `withConfig` decoration, before the connect chain runs. The enriched context flows through to the Session via the existing captured-request pattern.

### osc52gate scanner

The scanner is a state machine with these states:

```
normal → "esc" → "esc_bracket" → "num_5" → "num_52" → "semi_1" → "sel" → "semi_2" → "data" → "bel_or_esc" → "st_backslash"
```

Transitions emit or suppress bytes based on the mode. In `ModeDeny`, bytes are buffered once the escape is detected and suppressed on completion; on malformed sequences, the buffered bytes flush through. In `ModeAudit`, everything passes through; on completion, a log entry is emitted.

Implementation lives in an unexported `scanner` type in `osc52gate.go`, written as `io.Reader` (wraps the underlying reader and transforms on Read). Stateful across Read boundaries is natural in that shape.

### Testing strategy per middleware

- **osc52gate**: table-driven scanner test with a corpus of single-read, split-across-read, nested, and malformed inputs. Verify allow pass-through, deny strips, audit logs with correct fields. Independent scanner unit test plus a middleware-level integration that uses a fake Session.
- **idletimeout** (in `serve/builtins_test.go`): inject a controllable clock (interface with `AfterFunc` or a ticker). Verify inbound Write resets the timer; outbound Read does not; timer fire calls Close; `sess.Done()` stops the watchdog. Test concurrent goroutine lifecycle with a goroutine-count check.
- **recover**: handler panics → verify log entry contents, verify the panicModel's Init/Update/View behavior. Also verify a non-panicking handler passes through unchanged.
- **logging**: capture slog output via a test handler; verify start/end with correct duration and remote_addr. Use a fake Session with a context that has RemoteAddr set.
- **sipmetrics**: fresh `prometheus.Registry` per test. Verify gauge inc/dec on wrap/Close; verify duration histogram observation at Close; verify byte counters across Writes/Reads; verify errors counter on Close error.
- **Consolidation regression**: the existing `TestWSE2E_IdleTimeoutClosesSession` passes unchanged after `attachIdleTimeout` is replaced with auto-installed idletimeout middleware. Add one test that installs a user-supplied `idletimeout.New()` alongside `cfg.IdleTimeout` to verify both paths compose.
- **InitialResizeTimeout**: e2e test where the client connects but never sends the initial Resize; verify connection closes after the configured timeout without a session being created.

## Failure Modes

- **osc52gate scanner encounters malformed escape** (partial ESC ] 52 that never completes): buffered bytes flush through verbatim after a timeout or when a terminating byte is observed without proper context. Scanner does not drop data outside OSC 52 context.
- **idletimeout timer fires during an active inbound write**: the `sess.Close()` call races with the concurrent write. Close is idempotent (v0.3 contract), and InputWriter.Write on a closed session returns an error that the caller already handles. No special coordination needed.
- **recover'd handler runs on a tea.Program immediately**: panicModel.Init returns tea.Quit; the program terminates within a single frame. Client sees the error message briefly then the session closes. Acceptable.
- **logging middleware and panics**: named returns make the deferred log entry always fire. The "end" event includes the duration up to the panic point. Panic propagates normally (unless recover middleware is also installed, in which case the panic is caught before logging's defer observes it — order-sensitive).
- **sipmetrics and errors during Close**: the counter is incremented first, then the wrapped Close error is returned unchanged. Existing Session's Close idempotency contract applies.
- **InitialResizeTimeout fires**: close WS connection or WT stream; no session created; no error logged at default level (debug level only — misbehaving clients shouldn't spam server logs).

## Out of Scope for v0.4

- **osc52gate prompt mode** — requires a client-side event handler, a new Sip message type, and browser UI. Separate spec.
- **Per-Sip-message-type metrics** — requires a Sip-event hook in the core handlers. sipmetrics ships with byte-level granularity and session lifecycle; per-type counters can be added additively in a future patch without breaking changes.
- **Wish-shaped `activeterm`** — replaced by `Config.InitialResizeTimeout` because boba already enforces the dimension-validity invariant Wish's activeterm guards.
- **Built-in rate-limiter** — the design doc positions rate-limiting as a LiftHTTPMiddleware use case (wrap tollbooth or similar). No new middleware needed.
- **Middleware recipe documentation** — a `docs/RECIPES.md` enumerating common chains (e.g., `auth + conn-limit + osc52gate-deny + metrics + logging + recover`) would be useful but is separate from this spec.
